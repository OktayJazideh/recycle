گزارش حرفه‌ای RTL تهیه‌کننده: alireza.k

گزارش بازطراحی و ریفکتور کامل اپلیکیشن RecycleLink

این گزارش وضعیت بازطراحی UI/UX، بازسازی معماری Flutter، هم‌راستاسازی API و آماده‌سازی مسیر استقرار را جمع‌بندی می‌کند.

خروجی فعلی بر حذف داده‌های هاردکد، رفع برخورد با SafeArea، یکپارچه‌سازی تم، و تعریف مدل‌های داینامیک برای «نوع ضایعات» و «مقدار تقریبی» تمرکز دارد.

خلاصه پیشرفت

  • ورود به معماری جدید با Provider به‌عنوان تنها لایه مدیریت وضعیت انجام شد.
  • هدرها، اکشن‌بارهای پایین، فرم‌ها و کارت‌ها به ویجت‌های reusable منتقل شدند.
  • رنگ‌های تند نارنجی با پالت سبز/قهوه‌ای ملایم جایگزین شدند.
  • صفحه تیکت به ساختار keyboard-aware با MediaQuery.viewInsets منتقل شد.

کارهای باقی‌مانده

  • اجرای تحلیل و build نهایی Flutter در محیطی که Android SDK و Flutter SDK کامل داشته باشد.
  • اتصال سرویس بک‌اند نمونه به دیتابیس واقعی مانند PostgreSQL یا MySQL.
  • اضافه‌کردن placeholder assetهای نهایی داخل assets/images.
  • تبدیل placeholder screenهای سفارش و مقالات به ماژول‌های production-ready کامل.

ریسک‌ها و نکات

  • APK در این محیط ساخته نشد چون Flutter/Java/Android SDK حاضر نبود.
  • برای build قطعی باید یک runner لینوکسی/ویندوزی با toolchain کامل در اختیار باشد.
  • بخش backend فعلاً lightweight و in-memory است و برای دمو سریع مناسب است.

پیشنهادهای بهبود

  • افزودن repository جداگانه برای auth، requests، tickets و admin settings.
  • پیاده‌سازی persistence سمت موبایل با Hive فقط برای cache و نه منطق اصلی دامنه.
  • افزودن golden test برای کارت‌ها و snapshot test برای صفحات کلیدی.
  • افزودن analytics برای مسیرهای ثبت درخواست، پذیرش سفارش و تیکت پشتیبانی.

پروتکل تست سه‌مرحله‌ای

مرحله ۱: پیش‌بینی باگ

  • برخورد دکمه‌ها با نوار ناوبری در دستگاه‌های اندرویدی سه‌دکمه‌ای
  • sync نشدن قیمت‌های ویرایشی پنل مدیریت در فرم ثبت درخواست
  • کرش ناشی از نبودن داده category یا quantity هنگام bootstrap ناقص

مرحله ۲: شبیه‌سازی جریان کاربر

  • ورود ادمین ← ویرایش قیمت ← ذخیره ← مشاهده تغییر قیمت در فرم seller
  • ورود فروشنده ← ثبت درخواست فوری ← بازگشت به داشبورد ← بررسی کارت جدید
  • ورود جمع‌آور ← بازکردن تیکت ← بازشدن کیبورد ← ارسال پیام بدون پوشانده‌شدن input

مرحله ۳: لوپ سه‌باره رفع باگ

  1. Iteration A: اصلاح ساختار layout و padding پایه
  2. Iteration B: رگرسیون‌گیری روی موبایل کوچک، متوسط و تبلت
  3. Iteration C: تمیزکاری نهایی، حذف شرط‌های شکننده و ثبت fallbackها

آموزش گام‌به‌گام اجرا

  1. فایل .env.example را کپی کرده و مقدار API_BASE_URL را تنظیم کنید.
  2. برای بک‌اند سریع یکی از این دو را اجرا کنید: node backend/node/server.js یا python3 backend/python/server.py.
  3. در محیط دارای Flutter SDK، وابستگی‌ها را نصب کنید: flutter pub get.
  4. اپ را با متغیر محیطی API اجرا کنید: flutter run --dart-define=API_BASE_URL=http://127.0.0.1:8080.
  5. برای بسته‌بندی سورس، اسکریپت scripts/package_release.sh 2.0.0 یا python3 scripts/package_release.py 2.0.0 را اجرا کنید.
