۱. مقدمه و خلاصه پروژه

RecycleLink (بازار بازیافت) یک اپلیکیشن موبایل دوطرفه برای خرید و فروش ضایعات است که افراد عادی (فروشندگان) را با جمع‌آوری‌کنندگان (وانت‌داران سیار و مراکز ضایعات) متصل می‌کند.

تکنولوژی‌ها

  • فریمورک: Flutter 3.35.4 + Dart 3.9.2
  • State Management: Singleton Service Pattern
  • ذخیره‌سازی: Hive + SharedPreferences
  • UI: Material Design 3 (RTL-optimized)
  • پلتفرم هدف: Android 8+ (API 26+) + Web

۴. آموزش راه‌اندازی

پیش‌نیازها

  • Flutter SDK 3.35+ (flutter --version)
  • Dart SDK 3.9+
  • Android Studio / VS Code
  • Git
  • دستگاه اندروید یا شبیه‌ساز (API 26+)

گام‌به‌گام

  1. کلون یا دانلود: 
    git clone [repository-url] recyclelink
cd recyclelink
  2. نصب وابستگی‌ها: 
    flutter pub get
  3. بررسی محیط: 
    flutter doctor -v
  4. اجرا روی شبیه‌ساز/موبایل: 
    flutter run
  5. اجرا روی وب: 
    flutter run -d chrome
  6. ساخت APK نهایی: 
    flutter build apk --release
    خروجی: build/app/outputs/flutter-apk/app-release.apk
نکته: اپلیکیشن در حالت MVP با دیتای نمونه (Sample Data) اجرا می‌شود. برای اتصال به بک‌اند واقعی، فایل app_data_service.dart را با سرویس API جایگزین کنید.

۵. راهنمای پنل مدیریت

پنل مدیریت به صورت یکپارچه درون اپلیکیشن قرار دارد و از طریق دکمه «دمو مدیر» قابل دسترسی است.

ورود به پنل

  • در صفحه ورود، دکمه «مدیر» را بزنید
  • یا با شماره 09120000000 وارد شوید

بخش‌های پنل

بخشقابلیت‌ها
داشبوردآمار کلی: تعداد کاربران، سفارشات امروز/هفته/ماه، ارزش تراکنش‌ها، تیکت‌های باز
مدیریت کاربرانلیست فروشندگان و جمع‌آوری‌کنندگان، جستجو، فعال/غیرفعال کردن، مشاهده کامنت‌های مخفی
مدیریت سفارش‌هالیست تمام سفارش‌ها با فیلتر وضعیت، جستجو، تغییر وضعیت دستی
کامنت‌های مخفیمشاهده تمام کامنت‌ها با جستجو، امتیازات، حذف کامنت
مدیریت مقالاتافزودن، ویرایش، حذف مقاله، تغییر وضعیت انتشار
تیکت‌های پشتیبانیلیست تیکت‌ها با فیلتر، چت با کاربر، تغییر وضعیت تیکت
تنظیماتحداکثر فاصله جستجو، کارمزد تراکنش، حالت دمو، قیمت پایه ضایعات
گزارش‌گیریخروجی CSV کاربران، سفارش‌ها، تیکت‌ها

۶. چک‌لیست تست‌ها

تست‌های بخش فروشنده (۷ تست)

  • ثبت‌نام با شماره موبایل و OTP (کد 1234) - موفق
  • ورود با دمو فروشنده - بدون ثبت‌نام وارد می‌شود
  • ثبت درخواست با تمام فیلدها (نوع، وزن، عکس، آدرس، زمان، اولویت، پرداخت) - درخواست در لیست ظاهر می‌شود
  • دریافت پیشنهاد از جمع‌آوری‌کننده و قبول پیشنهاد - سفارش ایجاد می‌شود
  • پیگیری وضعیت سفارش با تایم‌لاین - تا مرحله تکمیل
  • امتیازدهی ۴ معیاره و نوشتن کامنت مخفی - ثبت موفق
  • ارسال تیکت پشتیبانی و مشاهده پاسخ ادمین - عملکرد صحیح

تست‌های بخش جمع‌آوری‌کننده (۵ تست)

  • ورود با دمو جمع‌آوری‌کننده - عملکرد صحیح
  • دیدن درخواست‌های نزدیک در لیست و نقشه - فیلتر فاصله فعال
  • ارسال پیشنهاد قیمت با پیام - ثبت موفق
  • تغییر وضعیت‌ها (در مسیر > تحویل > پرداخت نقدی > تکمیل) - جریان کامل
  • امتیازدهی به فروشنده - ثبت موفق

تست‌های پنل مدیریت (۱۰ تست)

  • ورود به پنل با دکمه مدیر - موفق
  • مشاهده داشبورد و آمار (۸ کارت آماری) - نمایش صحیح
  • غیرفعال کردن کاربر از لیست کاربران - وضعیت تغییر کرد
  • تغییر وضعیت سفارش به صورت دستی - عملکرد صحیح
  • مشاهده و حذف کامنت مخفی - موفق
  • افزودن مقاله جدید و مشاهده در اپ - نمایش داده شد
  • پاسخ به تیکت پشتیبانی و بستن تیکت - عملکرد صحیح
  • مشاهده گزارش تخلف (از طریق تیکت) - موجود
  • تنظیم حداکثر فاصله و کارمزد - ذخیره شد
  • خروجی CSV کاربران، سفارش‌ها، تیکت‌ها - داده‌های معتبر

۷. جریان کامل سفارش

  1. فروشنده وارد اپ می‌شود (دمو فروشنده)
  2. ثبت درخواست: نوع «کاغذ»، وزن ۲۰ کیلو، عکس، آدرس، امروز، پرداخت نقدی
  3. وضعیت: در انتظار پیشنهاد
  4. جمع‌آوری‌کننده درخواست را می‌بیند و پیشنهاد ۵۰,۰۰۰ تومان می‌دهد
  5. فروشنده پیشنهاد را قبول می‌کند → وضعیت: پذیرفته شده
  6. جمع‌آوری‌کننده «در مسیر» → در مسیر
  7. رسیدن و تحویل → «تحویل گرفته شد»
  8. پرداخت نقدی → وضعیت: تکمیل شده
  9. هر دو طرف امتیازدهی و کامنت مخفی ثبت می‌کنند
  10. فروشنده تیکت پشتیبانی ارسال می‌کند → ادمین پاسخ می‌دهد → کاربر پاسخ را می‌بیند

۸. نکات امنیتی و بهینه‌سازی

  • اعتبارسنجی ورودی: تمام فیلدهای فرم قبل از ارسال اعتبارسنجی می‌شوند
  • RTL Support: تمام صفحات برای زبان فارسی بهینه شده‌اند
  • Performance: استفاده از SingleChildScrollView و ListView.builder برای بهینه‌سازی
  • State Management: Singleton Pattern برای مدیریت متمرکز داده
  • Error Handling: نمایش SnackBar برای خطاها و موفقیت‌ها
  • HTTPS: در محیط تولید حتما از HTTPS استفاده شود
  • SQL Injection: با استفاده از ORM و Parameterized Queries جلوگیری شود
  • Token Management: استفاده از SecureStorage برای توکن‌های حساس

۹. اتصال درگاه پرداخت واقعی

برای جایگزینی شبیه‌ساز پرداخت با درگاه واقعی (مثلا زرین‌پال):

  1. ثبت‌نام در zarinpal.com و دریافت Merchant ID
  2. نصب پکیج: zarinpal: ^latest
  3. در فایل order_detail_screen.dart، متد _showPaymentSimulation را با کد واقعی جایگزین کنید: 
    // Replace _showPaymentSimulation() with:
Future<void> _startPayment() async {
  final response = await Zarinpal.request(
    merchantId: 'YOUR_MERCHANT_ID',
    amount: _order.finalPrice.toInt(),
    description: 'Payment for order ${_order.id}',
    callbackUrl: 'https://yourapp.com/verify',
  );
  // Launch payment URL
  // Verify payment on callback
}
  4. تست در محیط Sandbox زرین‌پال قبل از انتشار
